# Execution Plan: Retirement Planner Core Feature Set

Date: 2026-04-29

## Objective
Define and execute a UK consumer-first retirement modeling MVP that is easy to use initially and supports progressive advanced features.

## Phases
1. Scope and assumptions
- Lock UK consumer audience and baseline assumptions (inflation, longevity, pension assumptions).
- Define feature tiers (A/B/C).

2. Competitor benchmark matrix
- Build matrix across RightCapital, eMoney, MoneyGuide, Boldin, ProjectionLab; include Voyant once verified.
- Classify features as baseline, differentiator, advanced.

3. Tiered product definition
- Tier A launch scope (simple core planning).
- Tier B advanced optional modules.
- Tier C deferred roadmap.

4. UX progressive disclosure
- Keep onboarding simple.
- Expose advanced controls contextually and only when needed.

5. Validation
- Map each launch feature to concrete user jobs-to-be-done.
- Benchmark representative scenarios for quality and usability.
- Resolve Voyant verification gap via direct documentation/demo.

## Included scope
- UK consumer planning with progressive complexity.

## Excluded from v1
- Advisor workflows, enterprise reporting, deep estate strategy, full integrations.
