# Planning Artifacts

This folder stores planning artifacts generated in this session.

## Files
- `design/screen-inventory.md` - Prototype screen list, goals, and review checklist
- `design/prototype-notes.md` - How to review the UI concept and what is intentionally deferred
- `design/prototype.html` - Standalone static prototype for early interface review
- `research/retirement-research-summary.md` - Competitor and Voyant research findings
- `research/uk-rules-and-assumptions.md` - UK pension, tax, and retirement modeling defaults for v1
- `research/stochastic-analysis-data.md` - Guidance on Monte Carlo, stress testing, and market data requirements
- `product/core-feature-set.md` - Tiered feature definition for the UK consumer retirement modeling app
- `product/feature-matrix.md` - Value vs complexity scoring and exact v1 cut
- `product/product-requirements.md` - PRD for Tier A and Tier B progressive-disclosure behavior
- `product/domain-model-and-calculation-engine.md` - Stack-agnostic design for planning entities and the deterministic engine
- `plans/execution-plan.md` - Step-by-step implementation plan
- `plans/implementation-backlog.md` - Epics, tickets, sequencing, and milestone structure for delivery
- `testing/benchmark-personas.md` - Stable planning personas for model and product validation
- `testing/golden-test-cases.md` - Golden deterministic test cases and invariants
- `testing/test-strategy.md` - Layered testing strategy and acceptance gate for the first engine
