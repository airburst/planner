# Planner New User Guide

Planner helps you model retirement plans with local data, phased income, and tax-aware projections.

## What you can do today

- Create a household plan with one person or a partner.
- Enter starting assets and income streams with different start ages.
- Set a yearly spending target.
- Run a deterministic projection.
- Review sustainability, tax, drawdown, and yearly outcomes.

## Before you start

You should have rough values for:

- Your current age and target retirement age.
- Partner details (optional).
- Current savings/investments.
- Expected pension income and State Pension start ages.
- Annual spending target in retirement.

Use estimates first. You can refine numbers later.

## First run: create your first plan

1. Open Planner and go to the home page.
2. Select **Create New Plan** to launch onboarding.
3. Complete the 5 onboarding steps:
   - Household setup
   - Retirement timing
   - Current assets
   - Retirement income
   - Spending target
4. Finish onboarding to create the plan and open plan detail.

## Reading your plan detail

On the plan detail page, focus on these sections in order:

1. **Projection summary**
   - Quick sustainability status.
   - End-of-horizon asset balance.
   - Total tax and drawdown over the modeled period.
2. **Income phase chart**
   - Stacked annual income view by source.
   - Helps you spot bridge years before State Pension starts.
3. **Recommendations**
   - Priority-ranked actions with rationale.
   - Start with high-priority items first.
4. **Year-by-year table**
   - Detailed annual income, tax, effective rate, drawdown, and assets.

## How to iterate effectively

Use a small loop:

1. Change one assumption at a time.
2. Re-run and compare summary + recommendations.
3. Check bridge years and tax jumps in the table.
4. Keep notes on what improved sustainability most.

## Practical tips

- Keep early inputs simple; precision can come later.
- If the plan looks unsustainable, reduce spending or adjust retirement age first.
- In partner mode, review per-person effects because tax is calculated per person.
- Treat recommendations as prompts for exploration, not automatic advice.

## Current scope and limits

- Planner is deterministic today (not Monte Carlo).
- Data is local-first and stored on your machine.
- This is planning software, not regulated financial advice.

## Troubleshooting

- If projection data fails to load, retry from the error state in plan detail.
- If results look wrong, re-check ages and income start ages first.
- If numbers are unexpectedly high/low, verify annual vs monthly assumptions in your inputs.

## Next step after your first plan

Create a second plan with one major change (for example, retirement age or spending target), then compare outcomes manually to identify the highest-impact levers.
